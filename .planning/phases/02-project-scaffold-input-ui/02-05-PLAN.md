  <action>
Audit each file against the UI-SPEC Layout Contract and fix any deviations:

1. **Dark mode detection** — verify `src/App.tsx` or `src/main.tsx` sets `class="dark"` on `<html>` element based on `window.matchMedia('(prefers-color-scheme: dark)')`. If missing, add:
   ```typescript
   useEffect(() => {
     const mq = window.matchMedia('(prefers-color-scheme: dark)')
     const handler = (e: MediaQueryListEvent) => {
       document.documentElement.classList.toggle('dark', e.matches)
     }
     document.documentElement.classList.toggle('dark', mq.matches)
     mq.addEventListener('change', handler)
     return () => mq.removeEventListener('change', handler)
   }, [])
   ```

2. **WizardShell responsive layout:**
   - Mobile (< 640px): `px-4` (16px horizontal padding), no Card wrapper, `py-6` (24px vertical).
   - Desktop (>= 640px): `max-w-[600px] mx-auto`, Card wrapper with `rounded-lg border p-6`.
   - Navigation buttons: mobile `flex flex-col gap-3` (stacked, Next first), desktop `flex flex-row justify-between` (Back left, Next right).
   - All buttons: `min-h-[44px]` for touch targets.

3. **StepIndicator responsive:**
   - `overflow-x-auto` on the row container.
   - Labels: `<span className="hidden sm:inline">` so they hide below 640px.
   - Circles: 32px diameter, `min-w-[32px] min-h-[32px]`.
   - Each step item: `min-h-[44px]` touch target.

4. **SkillLevelSelector responsive:**
   - Cards full-width: `w-full`.
   - Card min height: `min-h-[72px]`.
   - Text left-aligned, padding `p-4`.

5. **MeasurementField responsive:**
   - Input: `w-full`.
   - Guidance text: max-width matches input width (no fixed max-width needed if both are `w-full`).
   - Label text: 14px minimum.

6. **PhysicalStep Select:**
   - Handlebar type Select: `w-full` on mobile.

7. **Dark mode color verification** — check all components use Tailwind dark: variants consistent with UI-SPEC:
   - Page background: `bg-white dark:bg-zinc-950`
   - Card/section backgrounds: `bg-zinc-100 dark:bg-zinc-900`
   - Text primary: `text-zinc-950 dark:text-zinc-50`
   - Text secondary: `text-zinc-500 dark:text-zinc-400`
   - Text muted: `text-zinc-400 dark:text-zinc-600`
   - Borders: `border-zinc-200 dark:border-zinc-800`
   - Focus ring: `focus-visible:ring-orange-500` (same in both modes)

8. **Font family** — verify `font-sans` on the root element resolves to Inter. If shadcn init set the font-family CSS variable, verify it includes Inter. If not, add to `src/index.css`:
   ```css
   :root {
     --font-sans: 'Inter', ui-sans-serif, system-ui, sans-serif;
   }
   ```
   (Or use `@theme` block if Tailwind v4 requires it.)

9. Run `npm run build` to verify all changes compile.
