<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Deploy to Vercel and verify public URL</name>

  <what-built>
    A complete Vite + React SPA that:
    - Accepts measurements across four input steps
    - Calculates Zwift Ride target positions using the calculation engine
    - Renders a four-step adjustment guide (saddle height, fore/aft, bar height, bar reach)
    - Persists all entered data to localStorage and restores on next visit
    - Provides a "Start over" button that clears localStorage and resets the wizard
  </what-built>

  <how-to-verify>
    **Option A — Vercel GitHub Integration (recommended, zero-config):**

    1. Push all code to a GitHub repository (create one if needed):
       ```
       git init && git add . && git commit -m "feat: complete zwift ride fitting assistant v1"
       git remote add origin https://github.com/YOUR_USERNAME/zwift-ride-fitting-assistant.git
       git push -u origin main
       ```
    2. Go to https://vercel.com → New Project → Import Git Repository
    3. Select the repository. Vercel auto-detects Vite and sets:
       - Build command: `npm run build`
       - Output directory: `dist`
    4. Click Deploy. Wait for build to complete (typically 60-90 seconds).
    5. Click the generated URL (e.g. `https://zwift-ride-fitting-assistant.vercel.app`)

    **Option B — Vercel CLI (no GitHub required):**
    ```bash
    npm install -g vercel
    vercel --prod
    ```
    Follow the prompts. Vercel will ask to link to an existing project or create new.

    **Verification steps after deployment:**

    Step 1 — Basic load:
    - Open the public URL in a browser
    - Confirm the page title "Zwift Ride Fitting Assistant" appears
    - Confirm the skill level selector (Beginner / Pro) is visible

    Step 2 — Input flow:
    - Select a skill level
    - In Physical Measurements, enter: Saddle Height = 750
    - Navigate through remaining steps to Results
    - Confirm an AxisCard appears for "Saddle Height" showing "750 mm" (or crank-corrected value)

    Step 3 — Persistence:
    - Refresh the page (Cmd+R or F5)
    - Navigate back to Physical Measurements
    - Confirm the saddle height field still shows 750

    Step 4 — Mobile check:
    - Open the URL on a phone browser (or use browser DevTools → device simulation → iPhone 390px)
    - Confirm no horizontal scrolling
    - Confirm form fields are thumb-reachable (inputs visible without zooming)

    Step 5 — Start over:
    - Navigate to Results
    - Click "Start over"
    - Confirm the wizard resets to the skill selector (step 0)
    - Refresh the page
    - Confirm fields are empty (localStorage cleared)
  </how-to-verify>

  <resume-signal>
    Type "deployed: URL" (e.g. "deployed: https://zwift-ride-fitting-assistant.vercel.app") once all five verification steps pass, or describe any issues found.
  </resume-signal>
