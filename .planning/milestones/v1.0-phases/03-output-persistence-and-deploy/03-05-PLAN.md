<objective>
Fix blur-triggered Zustand sync in all four input step components so that field values reach the store on field blur, not only on Next navigation.

Purpose: Without this fix the Zustand persist middleware never sees newly typed values until "Next" is clicked, making localStorage persistence unreliable. Users who leave a step without clicking Next silently lose their data on page refresh.

Output: Modified PhysicalStep, BodyStep, FrameStep, FitReportStep — each using `getValues(fieldName)` inside `handleBlur` instead of reading from the stale `watch()` snapshot.
</objective>
