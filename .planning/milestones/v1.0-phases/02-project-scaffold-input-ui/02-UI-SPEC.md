## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label-icon gap, helper text indent |
| sm | 8px | Input internal padding (vertical), label-to-input gap, inline note gap |
| md | 16px | Default element spacing, card internal padding, field-to-field gap |
| lg | 24px | Section padding (within a wizard step), group-to-group gap |
| xl | 32px | Wizard step content area padding (horizontal on mobile), step-to-step transition |
| 2xl | 48px | Major section breaks (between step indicator and first field) |
| 3xl | 64px | Page-level vertical breathing room (top/bottom of wizard) |

Exceptions:
- Touch targets for tappable step indicators and buttons: minimum 44px height (iOS HIG requirement; UX-03 mobile usability)
- "How to measure" expandable regions: 12px top/bottom padding inside the expanded panel (not on the 8-point grid; chosen for content density)

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Field help text, guidance paragraph text, section sub-descriptions |
| Label | 14px | 600 (semibold) | 1.4 | Form field labels, step indicator labels, skill selector option labels |
| Heading | 20px | 600 (semibold) | 1.2 | Wizard step titles (e.g. "Physical Measurements"), section group headers |
| Display | 28px | 600 (semibold) | 1.15 | Wizard overall title ("Zwift Ride Fitting Assistant"), result placeholder heading |

Font family (all roles): `Inter, ui-sans-serif, system-ui, sans-serif`

Notes:
- Only two weights are used: 400 and 600. No 500, 700, or other intermediates.
- The 14px label is the minimum font size used anywhere in the UI. Nothing smaller.
- Monospace (for mm values inline in guidance text): `ui-monospace, SFMono-Regular, Menlo, monospace` at 14px weight 400.

## Interaction Contracts

### Wizard Navigation

- Step indicator is a horizontal row of 4 tappable items: numbered circle + label. All steps are always tappable (D-01, D-06).
- Active step: orange-500 circle, semibold label.
- Completed step: zinc-500 circle with checkmark icon (lucide `Check`, 16px), muted label.
- Upcoming step: zinc-300 circle, muted label.
- "Next" and "Back" buttons sit at the bottom of each step content area. "Next" is filled accent button. "Back" is ghost variant.
- Step transitions: no animation for Phase 2. Simple conditional render swap. (Animation can be added in Phase 3.)

### Skill Level Selector

- Shown as the very first screen before the wizard steps (D-07).
- Two large tappable option cards (full width, stacked), not a dropdown.
- Selected option: orange-500 border (2px), faint orange-500/10 background tint.
- Unselected option: zinc-200 border, white/zinc-950 background.
- Labels: "I'm new to bike fitting" and "I know my measurements".
- Sub-labels (body text, 14px zinc-500): "I'll see measurement tips as I go" / "I'll keep tips collapsed".
- Selection persists to Zustand immediately on tap (D-10).

### Measurement Input Fields

- All measurement inputs accept numeric values only. Type: `number`, `inputMode="numeric"`, `min="0"`.
- Unit suffix "mm" displayed inline to the right of the input using a shadcn Input with a right-aligned unit slot or an `<Input>` wrapped with an absolute-positioned `mm` label. Do not use placeholder text for units — use a visible suffix element.
- Placeholder text shows the default value where applicable: `"172.5"` for crank length (D-14), `"40"` for hood height offset, `"50"` for hood reach offset.
- Validation timing: on blur only (D-Claude's Discretion). No inline error while typing.
- Validation errors: red-600 border, red-600 error message below field (14px, weight 400). Error copy follows the Copywriting Contract below.
- No fields are required (D-04). Empty = null in the store. Never show "required" asterisks.

### "How to Measure" Guidance (UX-01)

- Beginner mode: first field in each section has guidance auto-expanded on mount (D-08). Remaining fields show a `Collapsible` with trigger text "How to measure ▾" (zinc-500, 14px, lucide `ChevronDown` icon).
- Pro mode: all guidance collapsed by default. Each field label has a lucide `Info` icon (16px, zinc-400) inline after the label text. Clicking the icon expands the same `Collapsible` (D-09).
- Expanded guidance content: 16px body text, zinc-600 in light / zinc-400 in dark. Line height 1.5. Max-width matches the input field width.
- Complex field guidance (saddle height, saddle setback, handlebar reach) includes explicit reference point callouts in bold 14px semibold inline: e.g. "measure from **center of BB axle** to **top of saddle**, not the rails" (D-12).
- Simple field guidance (crank length, inseam, frame stack) is one sentence (D-13).

### Drop Bar Offset Fields (D-22)

- Shown only when handlebar type = "drop". Rendered immediately below the handlebar type selector within the Physical Measurements step.
- Fields are labeled "Hood height offset" and "Hood reach offset".
- Helper text below each field: "Default 40mm — override if you've measured your own hoods." / "Default 50mm — override if you've measured your own hoods."
- Collapsed into a `Collapsible` section labeled "Drop bar hood offsets (advanced)" — expanded by default when handlebar type changes to "drop".

### "Overridden by fit report" State (D-16)

- When a physical field has a value AND the corresponding `fitReport` field also has a value, the physical field renders with:
  - Label color: zinc-400 (muted).
  - Input: disabled, zinc-100 background (light) / zinc-900 background (dark).
  - Below input: italic 12px zinc-400 text — "Overridden by your fit report". This is NOT an error; use no red color.
- This state is computed from Zustand store, not from form state.

### Fit Report Priority Banner (D-15)

- Persistent amber-tinted info banner at the top of the Fit Report step.
- Background: `amber-50` (light) / `amber-950/30` (dark). Border: `amber-200` (light) / `amber-800` (dark).
- Icon: lucide `AlertCircle` in `amber-600`, 16px.
- Text (14px, zinc-700 light / zinc-300 dark): "Values here override your physical measurements for the same axis."
- This banner is not dismissable.

### Results Placeholder (D-20)

- Final wizard step (step 5). Displays a Card with:
  - Heading (28px display): "Your settings are ready"
  - Body (16px): "We've calculated your Zwift Ride target positions. Full adjustment guide coming in the next update."
  - Muted sub-line (14px zinc-500): "Come back soon — or check the project for updates."
- No output data is displayed in Phase 2.

## Layout Contract

### Mobile (< 640px — primary design target per UX-03)

- Single-column layout. Full viewport width with 16px (md) horizontal padding.
- Step indicator: horizontal scrollable row, circles 32px diameter, labels hidden on < 400px (show numbers only).
- Wizard card: no border-radius card chrome on mobile — full-bleed content with 24px vertical padding.
- Input fields: full width.
- Next/Back buttons: stacked column, "Next" first (top), "Back" second. Each button full width, 44px min height.

### Tablet and Desktop (≥ 640px)

- Max content width: 600px, centered.
- Step indicator: full labels visible.
- Wizard card: `Card` component with default shadcn border-radius (0.5rem), border, and `lg` (24px) padding.
- Next/Back buttons: inline row, right-aligned. "Back" ghost left, "Next" filled accent right. 44px min height.

### Wizard step order

1. Skill Level Selector (pre-wizard screen — not counted in step indicator)
2. Step 1: Physical Measurements (handlebar type selector, then all PHYS fields)
3. Step 2: Body Measurements (BODY-01, BODY-02, BODY-03)
4. Step 3: Frame Geometry (FRAME-01, FRAME-02, FRAME-03)
5. Step 4: Fit Report (FIT-01 fields with priority banner)
6. Step 5: Results (placeholder in Phase 2)
